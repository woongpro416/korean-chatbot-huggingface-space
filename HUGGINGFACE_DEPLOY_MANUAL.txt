Hugging Face Spaces 배포 매뉴얼
작성일: 2026-06-17
프로젝트: 개인 한글 챗봇

1. 업로드 대상 파일

Hugging Face Space에는 아래 4개 파일을 업로드한다.

- app.py
- requirements.txt
- Dockerfile
- README.md

아래 파일과 폴더는 업로드하지 않아도 된다.

- venv/
- __pycache__/
- main.py
- models/
- routers/
- services/
- data/
- HUGGINGFACE_DEPLOY_MANUAL.txt
- GITHUB_PORTFOLIO_MANUAL.txt

이 프로젝트의 Hugging Face 배포 버전은 app.py 단일 파일 기준으로 실행된다.
대화 히스토리와 피드백은 컨테이너 내부 SQLite 파일(data/chatbot.sqlite3)에 저장된다.
무료 Space는 재시작 시 저장 데이터가 초기화될 수 있으므로 장기 보관이 필요하면 Hugging Face Persistent Storage 또는 외부 DB를 검토한다.

2. 새 Space 생성

1) Hugging Face에 로그인한다.
2) 우측 상단 프로필 메뉴에서 New Space를 선택한다.
3) 기본 정보를 입력한다.

- Space name: 원하는 이름
- License: mit
- SDK: Docker
- Docker template: Blank
- Space Hardware: Cpu basic, 2 vCPU, 16 GB, Free

주의: SDK는 반드시 Docker를 선택한다.

3. 파일 업로드

1) 생성된 Space 상단의 Files 탭으로 이동한다.
2) Contribute 버튼을 누른다.
3) Upload Files를 선택한다.
4) app.py, requirements.txt, Dockerfile, README.md 파일을 업로드한다.
5) Commit changes to main 버튼을 눌러 저장한다.

4. 빌드 확인

파일을 커밋하면 Hugging Face가 자동으로 빌드를 시작한다.

- Building: 빌드 중
- Running: 실행 완료
- Runtime error: 실행 오류

첫 빌드는 Python 패키지 설치와 모델 다운로드 때문에 시간이 걸릴 수 있다.

5. 브라우저 테스트

Running 상태가 되면 상단 App 탭을 누른다.
채팅 화면에서 아래 문장으로 먼저 테스트한다.

- 안녕하세요
- 도움말
- 학습 기능이 있어?
- 인공지능을 쉽게 설명해줘

API 문서 화면은 Space 주소 뒤에 /docs를 붙여 접속한다.

예:
https://사용자명-스페이스명.hf.space/docs

Swagger UI 테스트 순서:

1) POST /chat 항목을 펼친다.
2) Try it out 버튼을 누른다.
3) Request body에 JSON을 입력한다.
4) Execute 버튼을 누른다.
5) Response body를 확인한다.

요청 예시:

{
  "session_id": null,
  "user_id": "anonymous",
  "message": "안녕하세요",
  "max_new_tokens": 160,
  "temperature": 0.6,
  "top_p": 0.9
}

6. 피드백과 학습 데이터

채팅 화면에서 챗봇 응답 아래 좋아요/싫어요 버튼을 누르면 /feedback API로 평가가 저장된다.
좋아요가 저장된 응답은 /training-data.jsonl에서 학습 데이터 형식으로 확인할 수 있다.

7. 수정 후 재배포

파일을 수정한 뒤에는 수정된 파일만 다시 업로드해도 된다.

- app.py만 수정: app.py만 업로드
- README.md만 수정: README.md만 업로드
- requirements.txt 수정: requirements.txt도 업로드
- Dockerfile 수정: Dockerfile도 업로드

상태가 헷갈릴 때는 4개 파일을 모두 다시 업로드해도 된다.

8. 자주 보는 오류

1) App 탭에 로그만 보이는 경우
- 빌드 또는 실행이 아직 끝나지 않은 상태일 수 있다.
- 잠시 기다린 뒤 새로고침한다.
- Running 상태인지 확인한다.

2) Runtime error가 뜨는 경우
- Logs 탭에서 마지막 오류 메시지를 확인한다.
- requirements.txt에 필요한 패키지가 빠졌는지 확인한다.
- Dockerfile의 CMD가 python app.py인지 확인한다.

3) 응답이 이상한 경우
- 현재 기본 모델은 Qwen/Qwen2.5-0.5B-Instruct이다.
- 가벼운 instruction 모델이므로 대형 상용 모델 수준의 정확도는 기대하기 어렵다.
- 인사, 도움말 같은 기본 문장은 app.py의 규칙 응답으로 처리한다.
