GitHub 포트폴리오 업로드 매뉴얼
작성일: 2026-06-17
프로젝트: 개인 한글 챗봇

1. 목적

이 저장소는 FastAPI, SQLite, Hugging Face Transformers, Docker를 사용한 개인 한글 챗봇 프로젝트를 포트폴리오로 정리하기 위한 저장소이다.

포트폴리오에서 보여줄 핵심 포인트:

- FastAPI 기반 웹 애플리케이션
- Hugging Face instruction/chat 모델 연동
- SQLite 기반 대화 히스토리 저장
- 사용자 피드백 저장
- 긍정 피드백 기반 training-data.jsonl export
- Docker 기반 Hugging Face Spaces 배포
- 브라우저 채팅 UI와 REST API 동시 제공

2. GitHub에 올릴 파일

GitHub 저장소에는 아래 파일을 포함한다.

- app.py
- requirements.txt
- Dockerfile
- README.md
- .gitignore

선택적으로 포함할 수 있는 파일:

- LICENSE
- screenshots/

올리지 않을 파일:

- venv/
- __pycache__/
- .env
- data/
- 모델 캐시 폴더
- 개인 토큰 또는 비밀번호가 들어간 파일
- HUGGINGFACE_DEPLOY_MANUAL.txt
- GITHUB_PORTFOLIO_MANUAL.txt

4. GitHub 저장소 생성

1) GitHub에 로그인한다.
2) New repository를 클릭한다.
3) Repository name을 입력한다.

추천:
korean-chatbot-huggingface-space

4) 포트폴리오용이면 Public을 권장한다.
5) Create repository를 클릭한다.

5. 로컬 프로젝트 Git 초기화

PowerShell에서 프로젝트 폴더로 이동한다.

cd C:\jwdev\chatbot

Git 저장소를 초기화한다.

git init
git add .
git commit -m "Build production-style Korean chatbot"

GitHub 원격 저장소를 연결한다.

git remote add origin https://github.com/사용자명/korean-chatbot-huggingface-space.git
git branch -M main
git push -u origin main

7. 포트폴리오 설명 예시

프로젝트명:
Hugging Face 기반 개인 한글 챗봇

설명:
FastAPI와 Hugging Face Transformers를 사용해 만든 개인 한글 챗봇입니다.
Docker 기반으로 Hugging Face Spaces에 배포할 수 있으며, 브라우저 채팅 UI, REST API, SQLite 대화 저장, 사용자 피드백 저장, 학습 데이터 export 기능을 제공합니다.

기술 스택:
Python, FastAPI, Pydantic, SQLite, Transformers, PyTorch, Docker, Hugging Face Spaces

개선 계획:

- 인증 적용
- 관리자용 피드백 대시보드
- RAG 기반 개인 문서 질의응답
- LoRA fine-tuning 파이프라인
- 테스트 코드와 GitHub Actions CI
